This folder contains a collection of microbenchmarks. Each subfolder contains a top program written in C++ and an upstream program written in Python

This microbenchmark stresses the event creation by sending events to multiple event queues. These events will create a number of events to the same queue. This way, each event queue should be stressed equally.

No multiprocessing on top, and no multi up down support yet\hypertarget{md___users_josemonsalvediaz__documents__project38_llvm_updown_ubenchmarks__r_e_a_d_m_e_autotoc_md28}{}\doxysection{DRAM Read Bandwidth}\label{md___users_josemonsalvediaz__documents__project38_llvm_updown_ubenchmarks__r_e_a_d_m_e_autotoc_md28}
Create multiple reads from different Up\+Down lanes. In order to saturate the bandwidth, each read message sends 64 bytes at once. This means, the return event contains 64/4 operands. It is possible to control the overall memory movement as long as it is a multiple of 64, and it is possible to control how many lanes will be part of the computation.
